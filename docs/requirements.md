# AI小说写作训练助手 - 需求文档

## 一、项目概述
项目名称：AI小说写作训练助手网页
开发目的：为小说写作者提供智能辅助工具，解决灵感断裂、语句不通、伏笔管理混乱等问题

## 二、用户故事

### 用户故事1：智能续写
作为一名小说作者，我可以打开写作页面的一个空白编辑器，输入任意已写内容，点击"智能续写"按钮，AI会根据当前光标位置的上文，生成3种不同风格的续写建议，我可以选择接受其中一种或继续修改。

### 用户故事2：伏笔记录与提醒
作为一名长篇作者，我希望系统能够自动记录我在写作中埋下的伏笔（如"主角小时候怕水""桌上有一把生锈的钥匙"），并在后续写作中当我提到相关场景时，侧边栏主动提醒我"有未使用的伏笔，是否需要呼应？"

### 用户故事3：纠错检查
作为一名对文笔有要求的作者，我可以在完成一段文字后点击"纠错检查"，系统会高亮疑似语法错误、逻辑矛盾、标点误用的地方，并给出修改建议，我可以一键应用或忽略。

### 用户故事4：主题练习模式
用户可在新建文档时选择一个内置主题（如"雨夜推理""初次相遇"），AI会生成一段200字以内的场景引导语。用户在引导语后写作，AI可针对该主题提供风格一致性检查。

## 三、不做什么（Out of Scope）
- 不做文字/作品的公开上传、发布或分享
- 不做用户之间的评论、点赞、私信等社交功能
- 不做富文本编辑器（仅支持纯文本或Markdown）
- 不做图片/附件上传
- 不做AI生成完整文章（只做辅助续写和建议）
- 不做多人协作或版本对比

## 四、约束条件
- 时间：7周
- 工具：Next.js + TailwindCSS + Supabase + AI API
- 数据：所有用户写作内容仅保存在该用户的私有存储中，不公开可见
- 隐私：用户密码哈希存储；不采集真实姓名、邮箱等个人信息；AI API调用时仅发送必要的上下文（最多2000字符）

## 五、验收标准

### 1. 登录与写作区
- 未登录用户首次进入时弹出注册界面，设定密码后分配递增数字ID
- 登录后可看到空白写作区，且本地草稿自动保存（每30秒），刷新页面不丢失

### 2. 智能续写
- 在写作区光标后输入至少50字上下文，点击"续写"按钮
- 5秒内返回3条不同风格的续写候选（每条不超过150字）
- 点击任一候选可插入写作区光标位置

### 3. 纠错检查
- 选中一段文字（或全选）后点击"纠错"
- 系统在10秒内返回错误列表（如"疑似语序不通""主语缺失""前后时态不一致"）
- 点击建议可替换原句

### 4. 伏笔记录与提醒
- 系统自动解析用户已写内容，抽取出明显伏笔（关键词引导）
- 存入侧边栏"伏笔箱"
- 当用户输入相关词汇时，侧边栏主动高亮该伏笔并附原文链接

### 5. 主题练习模式
- 用户可选择内置主题
- AI生成200字以内的场景引导语
- 提供风格一致性检查

### 6. 性能与稳定性
- AI响应时间平均不超过8秒
- 错误率低于5%
- 数据可完整备份恢复
